# Contributing to CodyFortran

First of all, thank you for considering contributing to CodyFortran! This is a small project, and any contributions are greatly appreciated.

## How to Contribute

There are several ways to contribute to the project:

*   **Reporting Bugs**: If you find a bug, please open an issue and provide as much detail as possible. This includes the version of CodyFortran you are using, the steps to reproduce the bug, and any relevant error messages.
*   **Suggesting Enhancements**: If you have an idea for a new feature or an improvement to an existing one, please open an issue to discuss it.
*   **Pull Requests**: If you have implemented a bug fix or a new feature, you can submit a pull request. Please make sure that your code follows the existing coding style and that you have tested your changes.

## Getting Started

If you are new to the project, a good place to start is by looking at the existing code and documentation. The `README.md` file provides a good overview of the project and its architecture. The `test/` directory contains several example programs that you can use as a starting point.

If you have any questions, please do not hesitate to open an issue.
